JSON Helper Guidance
  • /api/outputs/stream/<key> - Full live aircraft stream for the JSON output key.
  • /api/outputs/range/point/<lat>/<lon>/<radius_nm>?key=<key> - Aircraft inside a point/radius query (max 250 nm).
  • /api/outputs/json/<key>/v2/all - airplanes.live-style full dataset for this keyed output.
  • /api/outputs/json/<key>/v2/health - Compatibility health check for key and response timing.
  • /api/outputs/json/<key>/v2/hex/<hex>, /callsign/<callsign>, /reg/<reg>, /type/<type>, /squawk/<squawk> - Filter by aircraft identity fields.
  • /api/outputs/json/<key>/v2/mil, /ladd, /pia - Filter by common airplanes.live-style groupings.
  • /api/outputs/json/<key>/v2/point/<lat>/<lon>/<radius_nm> - Keyed point/radius path variant for compatibility clients.

Client certificate and private key

Get these from your TAK Server. When you are assigned to a group or channel, the server may give you a .p12 (or .pfx) file — that file contains both the certificate and the private key. You can upload the .p12 here (we extract cert and key), or upload two separate .pem files:

  • Certificate — the public client cert. Usually a .pem or .crt file; inside it says BEGIN CERTIFICATE. This identifies your connection to the server (e.g. which group/channel).
  • Private key — the secret key that pairs with the certificate. Usually a .pem or .key file; inside it says BEGIN PRIVATE KEY or BEGIN RSA PRIVATE KEY. Never share this file.

Stored encrypted; only you can add or replace. No one can view the files after upload.

TAK Server admin: The certificate must be assigned to the correct group (UserManager certmod -g) and that group must be in the data feed’s <filtergroup> in CoreConfig.xml — otherwise messages can be dropped. See project COT_PUSH_COMPLIANCE.md.

Use one option only — either cert + key (two files) or a single .p12 file.

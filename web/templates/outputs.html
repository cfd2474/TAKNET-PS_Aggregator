REST API (v2)

Airplanes.live–compatible · public, no auth

This aggregator exposes an ADS-B REST API at /v2/ with the same endpoint structure and response format as airplanes.live. Use it to query merged aircraft (local + ADSBHub) by hex, callsign, point/radius, and more.

Example: curl "https://adsb.tak-solutions.com/v2/point/33.8753/-117.5664/50" — aircraft within 50 nm of a point.

Endpoint Method Description
/v2/hex/<hex>GETAircraft with exact match on one or more Mode S hex IDs (comma-separated, limited to 1000)
/v2/callsign/<callsign>GETAircraft with exact match on one or more callsigns
/v2/reg/<reg>GETAircraft with exact match on one or more registrations
/v2/type/<type>GETAircraft with specified ICAO type code (e.g. A321, B738)
/v2/squawk/<squawk>GETAircraft squawking the specified value
/v2/milGETAircraft tagged as military
/v2/laddGETAircraft tagged as LADD
/v2/piaGETAircraft tagged as PIA
/v2/point/<lat>/<lon>/<radius_nm>GETAircraft within radius (nautical miles) of a point; radius up to 250 nm
/v2/allGETAll currently tracked aircraft
/v2/healthGETService health check

Responses use the same JSON envelope and field names as airplanes.live. For data field descriptions, see airplanes.live API guide and data field docs. Use a reasonable request rate (e.g. 1 request per second) when polling.

When disabled, the output is paused — no data is sent and API keys will not authenticate.

When enabled, configure transforms on the COTProxy config page (per-hex overrides).

Client certificate and private key

Get these from your TAK Server. When you are assigned to a group or channel, the server may give you a .p12 (or .pfx) file — that file contains both the certificate and the private key. You can upload the .p12 here (we extract cert and key), or upload two separate .pem files:

  • Certificate — the public client cert. Usually a .pem or .crt file; inside it says BEGIN CERTIFICATE. This identifies your connection to the server (e.g. which group/channel).
  • Private key — the secret key that pairs with the certificate. Usually a .pem or .key file; inside it says BEGIN PRIVATE KEY or BEGIN RSA PRIVATE KEY. Never share this file.

Stored encrypted; only you can add or replace. No one can view the files after upload.

TAK Server admin: The certificate must be assigned to the correct group (UserManager certmod -g) and that group must be in the data feed’s <filtergroup> in CoreConfig.xml — otherwise messages can be dropped. See project COT_PUSH_COMPLIANCE.md.

Use one option only — either cert + key (two files) or a single .p12 file.

Data filters
